# 纸质记录本使用规范

## 用途

介绍在断电、设备损坏或需要长期留痕时，如何使用纸质记录本记录事件、物资、健康、风险和决策，作为数字系统的低技术备份。

## 适用场景

适用于断电、设备不可用、外出采集、成员轮班、健康观察、物资盘点、风险记录和需要长期留痕的情况。

这篇内容用于让纸质记录成为数字系统的备用黑匣子。

## 操作步骤

### 核心原则

1. 纸质记录要简洁、连续、可追溯。
2. 每条记录必须有时间。
3. 重要记录要有地点和记录人。
4. 不要只写结论，要写观察依据。
5. 纸质记录应定期录入 Core 或拍照归档。

### 1. 编号

每本记录本编号：

```text
LB-NOTE-0001
```

首页写：

- 启用日期
- 使用人
- 用途
- 存放位置
- 页数

### 2. 连续书写

尽量按时间顺序写，不要到处跳页。

如果需要补记，应写明“补记”。

### 3. 标记重点

可用符号：

- `!` 表示风险
- `*` 表示重要
- `?` 表示待确认
- `✓` 表示已处理
- `→` 表示后续行动

### 4. 定期归档

每天或每周把重要记录：

1. 拍照。
2. 录入日记模块。
3. 标记纸质页码。
4. 放回固定位置。

## 判断标准

### 检查点

每周检查：

- 记录本是否在固定位置
- 是否受潮
- 是否缺页
- 是否有未归档重要记录
- 是否有关键事件没有时间
- 是否有成员看不懂的缩写
- 是否需要转录到 Core

## 风险提示

### 常见错误

- 只写“情况不好”，不写具体情况。
- 不写时间。
- 不写谁记录的。
- 记录本到处放。
- 只写在手机里，没有纸质备份。
- 写了但从不归档。
- 使用太多只有自己懂的缩写。

### 停止条件

出现以下情况时，应暂停使用并备份记录本：

- 记录本受潮
- 纸张破损
- 字迹开始模糊
- 涉及敏感成员信息
- 记录本可能丢失
- 内容需要长期保存

## 记录建议

### 推荐记录类型

包括：

- 日常事件
- 健康状况
- 物资消耗
- 设备故障
- 水电状态
- 外部风险
- 成员状态
- 决策记录
- 巡查记录
- 训练记录

### 基础格式

建议每条记录包含：

```text
日期：
时间：
地点：
记录人：
类型：
事件：
观察：
处理：
后续：
```

### 备注

壳中灯系统中，纸质记录本不是落后工具，而是低功耗、低依赖、抗系统故障的知识桥梁。

数字系统负责检索，纸质系统负责留痕。两者一起，才不容易断档。
