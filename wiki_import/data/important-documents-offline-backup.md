# 重要文件离线备份

## 适用场景

适用于家庭成员证件、医疗信息、联系方式、资产资料、设备说明和关键账户恢复信息的离线备份。

这篇内容用于在断网、设备损坏、人员分散或紧急撤离时保留基本身份和联络能力。

## 核心原则

1. 重要文件应有数字副本和纸质副本。
2. 敏感资料必须加密保存。
3. 只保存必要信息，避免无序堆放。
4. 备份应分级授权，不是所有成员都能看全部内容。
5. 定期检查资料是否过期。

## 建议备份内容

包括：

- 身份证件照片或扫描件
- 户口、护照、驾驶证等资料
- 医疗过敏史
- 常用药清单
- 慢性病记录
- 紧急联系人
- 家庭成员照片
- 重要设备说明
- 保险或资产摘要
- 离线地图关键点
- Core 恢复说明
- PocketBase 恢复说明

## 不建议明文保存

不建议明文保存：

- 完整银行卡密码
- 全部账户密码
- 私密聊天记录
- 不必要的敏感照片
- 未加密的完整成员档案

如果必须保存敏感信息，应使用加密容器或加密磁盘。

## 操作步骤

### 1. 分类

按类别建立目录：

```text
documents/
  identity/
  medical/
  contacts/
  devices/
  recovery/
  maps/
```

### 2. 命名

文件名应清楚但不过度暴露。

例如：

```text
member-A-medical-summary-2026-06.pdf
core-recovery-steps-2026-06.md
emergency-contacts-2026-06.pdf
```

### 3. 加密

敏感文件应：

1. 放入加密压缩包或加密磁盘。
2. 使用强密码。
3. 密码不与文件放在同一位置。
4. 给可信成员留下开启方式。

### 4. 纸质副本

纸质副本适合保存：

- 紧急联系人
- 药物过敏
- 常用药
- 血型
- 重要地址
- Core 启动步骤
- 设备恢复摘要

纸质副本应防水袋保存。

## 检查点

每季度检查：

- 证件是否过期
- 联系方式是否变化
- 药物信息是否更新
- 加密文件是否能打开
- 纸质副本是否受潮
- 是否有多余敏感信息
- 备份位置是否仍安全

## 常见错误

- 把所有证件照片明文放在桌面。
- 只保存电子版，没有纸质摘要。
- 密码忘了，备份打不开。
- 文件名混乱，紧急时找不到。
- 多年不更新联系人。
- 把敏感资料同步到不受控云端。

## 停止条件

出现以下情况时，应暂停扩散备份并重新整理：

- 加密密码泄露
- 备份设备丢失
- 成员权限变化
- 文件包含过多敏感信息
- 无法确认谁能访问
- 资料已经过期但仍在使用

## 备注

壳中灯后续成员系统应支持：

- 成员基础档案
- 医疗摘要
- 证件附件
- 紧急联系人
- 加密备份
- 分级访问

重要文件不是越多越好。真正有价值的是紧急时能找到、能打开、能证明、能行动。
