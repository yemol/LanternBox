# 离线地图包管理

## 用途

介绍离线地图包的下载、区域划分、版本记录、设备分发和定期检查，确保断网后仍能进行基础定位和路线判断。

## 适用场景

适用于断网、外出、灾害、长途移动、区域调查、避难路线规划和 Field Terminal 地图准备。

这篇内容用于确保地图不是“有网才有”，而是断网后仍能打开。

## 准备材料

### 地图范围

建议准备：

- 居住地周边
- 工作地周边
- 常走路线
- 医疗点周边
- 水源或补给点
- 避难点
- 城市出口路线
- 备用目的地

## 操作步骤

### 核心原则

1. 常用区域地图应提前离线保存。
2. 地图包要按区域和日期管理。
3. 多台设备应有必要地图副本。
4. 地图需要定期更新。
5. 关键路线和地点应有纸质或截图备份。

### 1. 划分区域

按区域命名：

```text
home-district-2026-06
work-district-2026-06
evac-route-north-2026-06
```

### 2. 下载离线包

确保下载完成后断网测试。

### 3. 记录版本

记录：

- 地图来源
- 下载日期
- 覆盖区域
- 文件大小
- 所在设备
- 是否测试可打开

### 4. 分发设备

至少分发到：

- 手机
- Field Terminal
- Core
- 备用存储
- 纸质摘要

## 判断标准

### 检查点

每月检查：

- 地图能否打开
- GPS 是否能定位
- 区域是否完整
- 是否缺少关键路线
- 是否有过期版本
- 备用设备是否有副本
- 是否能离线搜索地点

## 风险提示

### 常见错误

- 以为收藏地点等于下载地图。
- 下载后从未断网测试。
- 只存一台手机。
- 没记录地图版本。
- 地图包太大导致设备没空间。
- 关键路线没有纸质备份。

### 停止条件

出现以下情况时，应重新下载或调整地图包：

- 地图打不开
- 区域缺失
- 文件损坏
- 设备空间不足
- 路线发生重大变化
- 新设备没有地图
- 地图版本太旧

## 记录建议

### 备注

壳中灯后续地图模块应支持：

- 地图包清单
- 区域标签
- 设备分发状态
- 更新日期
- 关键路线备注

离线地图不是为了优雅导航，而是在网络黑掉时，还能知道自己站在哪一页世界上。
