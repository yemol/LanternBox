# 通信日志最小字段

## 用途

用于记录无线电、LoRa、短波收听、纸条和人工传递的最小字段。日志不是为了好看，而是为了交接、复盘和失联判断。

## 操作步骤

1. 每条记录写时间、记录人和通信方式。
2. 写设备编号、频道或节点编号。
3. 写消息级别：危险、位置、资源、普通或待核。
4. 写是否收到回执，以及谁确认。
5. 写失败原因、下次窗口和待处理动作。

## 判断标准

- 可继续：接手人能知道发生了什么、谁确认、下一步做什么。
- 需补记：只有“已联系”但没有时间、对象或结果。
- 停止条件：生命危险、成员失联、身份不清或未确认消息被当作已确认时，停止普通交接并复核日志。

## 风险提示

- 不要只记成功消息。
- 不要把待核短波信息写成事实。
- 不要让通信记录暴露完整住所和库存。
