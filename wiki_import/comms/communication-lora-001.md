# LoRa 节点适合传什么信息

## 用途

用于界定 LoRa 节点适合传递的信息类型，发挥低功耗、短文本、间歇覆盖的优势。

## 适用场景

- 团队有 LoRa 终端或节点，但带宽低、消息短、不能保证实时送达。
- 需要在局部区域传递状态、位置、需求或告警。
- 不适用于传输长文、地图图片、复杂诊疗记录或敏感完整名单。

## 准备材料

- LoRa 节点
- 短消息模板
- 节点编号表
- 电量记录
- 重发规则

## 操作步骤

1. 为每个节点设编号和常用位置，不用真实姓名作为唯一身份。
2. 消息控制在短字段：节点、时间、位置编号、状态、需求、下一窗口。
3. 重要信息重复发送有限次数，并记录是否收到确认。
4. 把长任务拆成编号和状态，不在 LoRa 上解释复杂背景。
5. 每日检查天线、电量、时间和最近一次成功收发。

## 判断标准

- 消息短、可解析、能离线记录，接收方知道下一步。
- 节点编号表与纸质地图和联系人表一致。
- 未确认送达的消息不会被当作已经执行。

## 风险提示

- 不要用 LoRa 发送完整住址、库存、人员名单和长期路线。
- 不要假设 LoRa 一定实时到达。
- 不要把天线损坏或低电量节点作为唯一通信方式。

## 替代方案

- 没有 LoRa 覆盖时，转为短信、对讲、纸条或固定集合点。
- 需要传长内容时，LoRa 只发索引和集合时间，细节用纸质交接。

## 记录建议

记录节点编号、消息格式、收发时间、确认状态、电量和失败原因。
