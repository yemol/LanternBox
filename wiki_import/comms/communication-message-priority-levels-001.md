# 紧急信息优先级

## 用途

用于通信窗口时间短、电量少或信道拥挤时决定先发什么。优先级让危险信息先通过，也让普通信息不挤占窗口。

## 操作步骤

1. 把生命危险、火源、受伤、失联和路线阻断列为最高优先级。
2. 第二级发送位置、状态和返回时间。
3. 第三级发送资源需求和任务交接。
4. 普通更新等窗口空闲再发。
5. 未核信息标为待核，不当作行动指令。

## 判断标准

- 可继续：每条消息能说出优先级和是否需要回执。
- 需调整：普通聊天、重复信息或猜测挤占窗口。
- 停止条件：生命危险消息未确认、身份不清或信道被占用时，暂停普通消息。

## 风险提示

- 不要把“听说”当作紧急事实。
- 不要用普通窗口讨论敏感资料。
- 不要让低优先级消息耗尽发射电量。
