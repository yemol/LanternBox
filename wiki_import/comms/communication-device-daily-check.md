# 通讯设备每日检查

## 适用场景

适用于手机、对讲机、LoRa 节点、短波电台、Field Terminal、学习终端和 Core 周边通讯设备的日常检查。

这篇内容用于确保通讯设备不是“理论上有”，而是当天真的能用。

## 核心原则

1. 通讯设备每天至少检查一次。
2. 电量、频道、时间和线材都要检查。
3. 设备状态要记录。
4. 不熟悉的设备不应等紧急时才第一次开机。
5. 重要通讯应尽量有备用路径。

## 检查项目

### 1. 电量

检查：

- 主设备电量
- 备用电池
- 移动电源
- 充电线
- 充电口
- 是否可以正常充电

### 2. 时间

检查：

- 系统时间是否正确
- 记录时间是否统一
- 多设备时间是否相差过大
- 是否需要手动校准

时间错误会影响日志、定位和事件复盘。

### 3. 频道与频率

检查：

- 对讲频道
- LoRa 设置
- 短波频率记录
- 呼号或身份标识
- 约定监听时间

不熟悉的频率和频道不要随意占用。

### 4. 天线与连接

检查：

- 天线是否接好
- 接头是否松动
- 线材是否破损
- 外接模块是否识别
- 有线同步是否可用

### 5. 记录

记录：

- 检查时间
- 设备名称
- 电量
- 是否能开机
- 是否能收发
- 当前设置
- 异常情况

## 操作步骤

1. 按设备清单逐个开机。
2. 查看电量。
3. 检查时间。
4. 检查频道或配置。
5. 做一次短消息或本地测试。
6. 检查线材和天线。
7. 记录状态。
8. 异常设备贴标签并隔离。

## 检查点

每天检查：

- 手机是否可用
- Field Terminal 是否可用
- 对讲设备是否可用
- LoRa 节点是否在线
- 充电线是否在
- 移动电源是否够电
- 约定频道是否一致
- 是否有新故障

## 常见错误

- 只看设备在不在，不开机测试。
- 频道被改了没人知道。
- 电量低但没有充。
- 天线没接就发射。
- 日志时间错误。
- 备用线材丢失。
- 多人使用同一设备但不交接状态。

## 停止条件

出现以下情况时，应停止使用或暂停发射：

- 天线未连接
- 设备异常发热
- 电池鼓包
- 接口烧蚀
- 频率或规则不清楚
- 设备进水
- 发送内容可能暴露敏感位置或成员信息

## 备注

壳中灯后续通讯系统应把每日检查形成固定表单。

推荐记录格式：

```text
日期：
设备：
电量：
频道：
时间：
收发测试：
异常：
处理：
```

通讯设备的价值不在“能喊出去”，而在“关键时刻能用正确方式喊出去”。
