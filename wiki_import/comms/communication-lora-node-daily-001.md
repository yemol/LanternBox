# LoRa 节点每日状态检查

## 用途

介绍 LoRa 节点在离线通信网络中的每日基础检查，包括电量、天线、时间、节点编号、收发测试和异常记录。

## 适用场景

适用于壳中灯 Field Terminal、Sensor Node、通信前哨节点和简易 LoRa 通信网络的日常维护。

这篇内容用于保证低速通信链路在需要时仍然可用。

## 准备材料

### 检查项目

### 1. 节点身份

检查：

- Device ID
- 节点名称
- 所属角色
- 最近登记时间
- 是否允许接入

### 2. 电量

检查：

- 电池电量
- 充电状态
- 备用电源
- 是否低电
- 是否异常发热

### 3. 天线

检查：

- 天线是否接好
- 接口是否松动
- 天线是否折断
- 是否被金属遮挡
- 是否进水

### 4. 收发测试

可进行低频测试：

- 发送测试消息
- 接收确认
- 记录延迟
- 记录失败次数
- 记录信号强弱

## 操作步骤

### 核心原则

1. LoRa 节点必须有编号。
2. 每日检查电量、天线和收发状态。
3. 通信记录要带时间。
4. 未登记设备不应接入核心网络。
5. 通信内容应遵守端到端加密原则。

1. 打开节点。
2. 检查电量。
3. 检查天线。
4. 确认时间。
5. 发送测试包。
6. 等待确认。
7. 记录结果。
8. 异常节点贴标签并排查。

## 判断标准

### 检查点

每日检查：

- 是否能开机
- 是否能发送
- 是否能接收
- Device ID 是否正确
- 时间是否正确
- 电量是否足够
- 是否有异常发热
- 是否有重复失败

## 风险提示

### 常见错误

- 天线没接就测试。
- 节点没有编号。
- 多个节点名字重复。
- 不记录失败。
- 长期低电运行。
- 让未知设备接入。
- 只测试发送，不测试接收。

### 停止条件

出现以下情况时，应停止节点运行或隔离：

- 电池鼓包
- 设备进水
- 天线损坏
- 异常发热
- Device ID 异常
- 加密校验失败
- 重复出现未知消息
- 无法确认设备来源

## 记录建议

### 备注

壳中灯通信原则要求：Core、Field Terminal、Study Terminal、Sensor Node 之间的通信都必须端到端加密。

LoRa 是慢信使，不是大水管。它的价值在于低功耗、远距离、可验证。
