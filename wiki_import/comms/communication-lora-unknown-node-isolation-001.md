# 未知 LoRa 节点隔离

## 用途

用于 LoRa 或 Mesh 网络中出现未登记节点、陌生编号、重复编号或身份不清设备时的处理。核心原则是先隔离，再核查。

## 操作步骤

1. 记录未知节点出现时间、编号、信号和消息内容。
2. 不把未知节点加入关键通信链。
3. 暂停向该节点发送位置、库存、病人和路线信息。
4. 核对节点清单、负责人和最近移动记录。
5. 确认前只允许低敏测试消息或完全静默。

## 判断标准

- 可恢复：负责人能确认节点来源、位置、用途和最近检查记录。
- 需隔离：编号重复、身份无人认领或内容要求敏感信息。
- 停止条件：未知节点引导移动、索要位置、反复发送异常消息或疑似暴露网络时，停止普通通信。

## 风险提示

- 不要因为信号强就信任节点。
- 不要把新设备直接接入主链路。
- 不要用真实库存和住址测试陌生节点。
