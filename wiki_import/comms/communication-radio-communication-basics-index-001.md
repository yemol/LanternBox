# 无线电通信基础索引

## 用途

用于在外部网络不可用、队伍需要低带宽联络或需要理解无线电设备限制时，帮助使用者判断应查频率、传播、天线、LoRa、短波还是通信纪律。它解决的是“该把通信问题转到哪个本地 Guide 或 Wiki”的导航问题，而不是直接保证联络成功。

## 适用场景

适用于准备通信计划、排查低带宽通信失败、比较无线电/LoRa/短波用途、制定消息确认规则时。不需要用于普通面对面交接、纸条传递已经足够、或设备没有电量和天线条件的情况。

## 准备材料

需要输入设备类型、可用电量、预定通信窗口、位置范围、消息重要性、是否有已知频段或呼号、已有通信记录。没有完整资料时至少记录设备、时间、地点、电量和要传的消息。

## 操作步骤

1. 明确当前问题是设备不会用、距离不够、收不到回应、消息太长，还是敏感信息不该发送。
2. 先查对应本地 Guide：通信计划、联络窗口、短信/无线电短消息、LoRa 或短波条目。
3. 再用本索引查背景概念，例如频率、传播、天线方向、低带宽确认机制。
4. 把查到的背景转成现场判断：能否缩短消息、调整时间窗口、改用中继人员或纸质交接。
5. 若涉及位置、库存、人员弱点等敏感信息，转入低暴露沟通规则，不继续扩大广播范围。

## 判断标准

- 是否需要继续查询：能把通信问题归入设备、电量、时间窗口、距离、消息格式或暴露风险中的一种，并找到对应 Guide/Wiki。
- 是否需要转入其他知识模块：只知道“无线电不好用”，但没有设备、电量、位置、时间和消息内容；或查到背景资料后无法转成行动。
- 是否停止依据：有明确问题和本地行动卡时继续查询；缺少设备状态、电量或消息目标时，先补记录再查。

## 风险提示

- 风险来源：把无线电当作稳定通信保证，忽略电量、确认机制和信息暴露。
- 错误理解可能导致的问题：关键消息未送达、过度耗电、暴露位置和库存、队伍误以为已经完成联络。
- 不确定时停止依据：无法确认接收方、消息必要性、电量余量或敏感信息边界时，停止扩大无线发送，改用更低暴露方案。

## 替代方案

缺少完整资料时只使用纸本通信计划和已知联络窗口；缺能源时优先保留短消息、预设暗号和人工传递路线；无设备时把本索引用作术语解释，不做无线方案。

## 记录建议

记录查询原因、设备、电量、频段或方式、通信窗口、最终转入的 Guide/Wiki、发送内容摘要、是否收到确认和下一次复查时间。
