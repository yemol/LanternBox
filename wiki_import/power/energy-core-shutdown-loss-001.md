# Core 设备断电前处理

## 用途

介绍在电量不足、计划断电、移动 Core 或供电不稳定时，对 Core 主机、PocketBase、模型服务和数据进行安全关闭与备份的步骤。

## 适用场景

适用于 Core 主机电量不足、移动电源即将耗尽、计划断电、搬动设备、供电不稳定或需要关闭系统保存数据的情况。

这篇内容用于降低数据损坏、数据库异常和系统无法启动的风险。

## 准备材料

原文未单列准备材料；执行前按下方操作步骤准备所需物品。

## 操作步骤

### 核心原则

1. 不要直接拔电。
2. 先停止写入，再关闭服务。
3. 重要数据先备份。
4. 记录关机原因和时间。
5. 重新开机后要做系统自检。

### 关机前检查

检查：

- 是否有人正在写日记
- 是否正在导入 Wiki
- 是否正在修改物资库存
- 是否正在生成语音
- 是否正在同步文件
- 是否有未提交代码
- 是否有未备份数据

### 1. 通知成员

如果多人使用系统，先说明：

- Core 即将关闭
- 不要继续提交数据
- 当前页面保存完毕
- 预计关闭时间

### 2. 保存数据

确认：

- Wiki 导入完成
- PocketBase 页面无未保存编辑
- 代码修改已保存
- 重要 Markdown 原稿已保存
- 日记和库存页面无未提交内容

### 3. 备份关键目录

优先备份：

- PocketBase 数据目录
- wiki_import
- emergency 数据
- app 目录
- scripts 目录
- 配置文件

### 4. 停止服务

按实际运行方式停止：

- FastAPI 服务
- PocketBase
- Ollama 或本地模型服务
- TTS 服务
- 其他后台脚本

终端中运行的服务可用 `Ctrl + C` 正常停止。

### 5. 正常关机

确认服务停止后，再进行系统关机。

不要在硬盘写入时强制断电。

## 判断标准

### 检查点

关机前确认：

- 数据已保存
- 服务已停止
- 关键目录已备份
- 没有正在写入任务
- 电源线和硬盘状态安全
- 关机原因已记录

## 风险提示

### 常见错误

- 直接拔电源。
- PocketBase 正在写入时断电。
- Wiki 导入一半中断。
- 以为网页关了就是服务关了。
- 没有记录关机原因。
- 开机后不做自检。

### 停止条件

出现以下情况时，应优先保护数据，不要继续普通操作：

- 电量低于安全阈值
- 外置硬盘异常
- 数据库正在导入
- 系统明显卡死
- 电源频繁闪断
- 有烧焦味或设备异常发热

## 替代方案

### 重新开机后

开机后执行：

1. 启动 PocketBase。
2. 启动 FastAPI。
3. 打开系统自检页。
4. 检查 Wiki、应急指南、库存、日记。
5. 检查 Ollama 和 TTS。
6. 查看是否有数据缺失。
7. 记录恢复时间。

## 记录建议

### 备注

壳中灯系统中，Core 断电前处理应逐步做成自动流程。

未来目标：

- 一键停止服务
- 一键备份
- 自动生成关机记录
- 下次启动自动提示上次关机状态

Core 是知识炉心，关机不是熄火，而是把火种妥善封进灯芯。
