# 低电量任务排程

## 用途

介绍 Core、终端和移动设备在低电量状态下如何安排任务优先级，优先保障通讯、记录、照明、数据保存和必要检索。

## 适用场景

适用于 Core 主机、Field Terminal、Study Terminal、手机、移动电源或照明设备电量不足，但仍需要维持基本运行的情况。

这篇内容用于帮助决定哪些任务继续执行，哪些任务暂停，避免把有限电力浪费在低优先级操作上。

## 操作步骤

### 核心原则

1. 低电量时先保命，再保信息，再保舒适。
2. 通讯、照明、时间、记录和数据保存优先。
3. 大模型、索引、图像分析和批量处理应延后。
4. 设备应从高耗能模式切换到低功耗模式。
5. 断电前应先保存数据和停止写入任务。

### 任务优先级

### 第一优先级：安全与通讯

包括：

- 发送必要消息
- 查看关键联络信息
- 保持最低照明
- 查看时间
- 记录紧急事件
- 保存正在编辑的数据

### 第二优先级：关键检索

包括：

- 应急指南检索
- Wiki 关键条目查询
- 物资库存查询
- 健康记录查询
- 地图关键点查看

这类任务应短时使用，查完即关闭屏幕或退出高耗能页面。

### 第三优先级：系统维护

包括：

- 数据库备份
- 关闭服务
- 保存日志
- 记录电量状态
- 准备断电恢复说明

如果电量很低，应优先做最小备份，而不是完整迁移。

### 第四优先级：延后任务

低电量时应暂停：

- 大模型长对话
- 批量生成内容
- 大量图片处理
- 长时间语音播报
- 全库索引
- 大文件复制
- 娱乐用途

### Core 低电量处理

Core 电量不足时：

1. 停止大模型任务。
2. 停止非必要 TTS。
3. 保存当前数据。
4. 关闭不必要网页和服务。
5. 执行必要备份。
6. 记录关机原因。
7. 正常关闭系统。

不要在电量即将耗尽时继续做长时间 AI 推理。

### 终端低电量处理

移动终端电量不足时：

1. 降低屏幕亮度。
2. 关闭不必要无线功能。
3. 缩短屏幕开启时间。
4. 只保留必要监听或记录。
5. 关闭连续定位，改为间歇定位。
6. 保留一部分电量用于求助和回传。

## 判断标准

### 检查点

进入低电量模式前检查：

- 当前电量
- 预计剩余时间
- 是否有移动电源
- 是否有太阳能输入
- 是否有未保存数据
- 是否有未完成通讯
- 是否需要备份
- 是否需要通知成员

## 风险提示

### 常见错误

- 低电量时继续长时间 AI 对话。
- 一边断电一边导入大量 Wiki。
- 忘记保存日记和库存修改。
- 把最后电量用于娱乐或长时间照明。
- 直接拔电导致数据库风险。
- 不记录关机原因，开机后无法复盘。

### 停止条件

出现以下情况时，应立即进入最低功耗或关机流程：

- 电量低于安全阈值
- 电源输入不稳定
- 移动电源即将耗尽
- Core 正在写入但电量不足
- 设备异常发热
- 需要保留电量用于夜间照明或通讯
- 无法确认下一次充电时间

## 记录建议

### 备注

壳中灯系统后续应建立能源面板，显示：

- 当前功耗
- 剩余电量
- 预计运行时间
- 高耗能任务
- 建议暂停项目
- 低电量任务队列

低电量不是系统变弱，而是系统进入火种模式。火种模式的目标不是做更多事，而是让关键能力活到下一次补能。
