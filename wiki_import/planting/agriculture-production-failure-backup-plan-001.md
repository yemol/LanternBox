# 生产失败备用计划

## 用途

让一季失败不会直接中断长期生产链。

## 适用场景

- 一批种子没出芽。
- 虫害毁掉主作物。
- 雨季导致地块积水。

## 准备材料

- 地块或批次标签
- 记录纸和笔
- 干净容器或隔离标记
- 必要时使用单独手套或工具

## 操作步骤

1. 为每个主作物写一个替代作物或补种窗口。
2. 保留少量种子和容器给备用批次。
3. 识别失败原因：种子、水、土、天气、病虫害或照护。
4. 只在原因清楚后扩大下一批。
5. 把备用计划写入种植日历。

## 判断标准

- 主作物失败后有下一步。
- 备用种子没有提前用光。
- 失败原因被记录。

## 风险提示

- 不把农业问题和入口食物、人身污染或工具维修混成同一判断；涉及食用、污染或人身症状时转入相应领域协同。
- 不使用农药配方、化学灭虫剂自制或高复杂农业工程作为默认方案。
- 停止条件：原因未明且可能涉及污染或病害时，不在同地块补播。
- 停止条件：连续两批失败时停止扩大规模。

## 替代方案

- 改用容器或不同地块小样。
- 优先短周期、低水量作物恢复产出。

## 记录建议

记录失败批次、判断原因、备用动作、补种日期和结果。
