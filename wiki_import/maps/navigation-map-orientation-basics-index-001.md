# 地图与方位基础索引

## 用途

用于在读地图、确认方向、选择地标、估算路线或交接路径时，帮助使用者判断应查比例尺、地标选择、返回路线、危险区域标记还是离线地图维护。

## 适用场景

适用于路线规划、取水、避难点比较、外出前复核和交班说明。不需要用于已知短路线、有人现场带路且无需记录的低风险移动。

## 准备材料

需要输入当前位置、目标点、已知地标、路线草图、地图版本、时间、天气、人员状态和返回点。最低条件是起点、方向、目标和一条可回撤路径。

## 操作步骤

1. 明确当前问题是看不懂方向、比例尺、地标、路线难度、返回点还是禁行区。
2. 先查具体 Wiki：地图比例尺、地标选择、返回路线标记、危险区域标记、离线地图维护。
3. 再用本索引补充方位、地图符号和方向概念。
4. 把背景知识转成路线草图、节点、返回时间和禁行点。
5. 如果现场与地图不一致，停止按旧图行动，转入环境变化复盘。

## 判断标准

- 是否需要继续查询：起点、目标、地标、方向、返回点和地图版本清楚，能转入具体路线 Wiki。
- 是否需要转入其他知识模块：地图过旧、方向不明、地标相似、天气或水位改变、人员无法复述路线。
- 是否停止依据：路线可回撤且能用具体条目确认时继续；方向、地标或地图版本不明时先补记录或停止外出。

## 风险提示

- 风险来源：把地图背景知识当成现场确认，忽略旧地图、方向误读和路线变化。
- 错误理解可能导致的问题：迷路、误入危险区、返回失败、交接误读或夜间行动风险扩大。
- 不确定时停止依据：无法确认起点、方向、返回点、地图版本，或现场标志与地图冲突时，停止按该索引直接行动。

## 替代方案

缺少地图时用地标、方向、步数和返回点画草图；缺能源时优先使用纸本和口头复述校验。

## 记录建议

记录查询问题、地图版本、起点目标、采用的具体 Wiki、路线节点、禁行点和复查时间。
