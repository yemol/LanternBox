# 任务路线记录

## 用途

用于把一次外出任务从临时行动变成可复用路线记录，支撑下次外出和 Field Terminal 轨迹复盘。

## 适用场景

- 取水、巡查、采集、探路或撤离前侦察。
- 路线可能被别人重复使用。
- 返回后需要更新危险点和耗时。

## 准备材料

- 任务表
- 路线卡
- FT track 或纸质记录
- 地图

## 操作步骤

1. 写明任务目的、人员、起止时间和路线编号。
2. 记录关键检查点、资源点、危险点和等待点。
3. 返回后补实际耗时、偏离点和失败点。
4. 把 FT track 中的关键点转成文字摘要。
5. 标记该路线是否适合复用或只作为历史记录。

## 判断标准

- 可复用：目的、路线、风险、耗时和返回线清楚。
- 需复查：路线依赖临时条件或危险点未确认。
- 停止条件：记录缺返回线或关键点不明时，不让第二人照走。

## 风险提示

- 不要只记录成功部分。
- 不要把一次晴天路线默认适用于夜间或雨后。

## 替代方案

- 没有 track 时用检查点和地标顺序补写。
- 无法复用时标为历史参考，不进入主路线库。

## 记录建议

记录任务编号、路线版本、执行人、复查人、可复用状态和下次注意点。
