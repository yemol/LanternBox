# LanternBox 已确认决策

## D001 - 默认离线

LanternBox 默认离线运行，不依赖互联网或云服务。

## D002 - 无外部支援假设

应急模式和极端环境设计中，默认没有外部救援、没有公网、没有现代公共服务。

## D003 - 所有设备通信必须端到端加密

Core、Field Terminal、Study Terminal、Sensor Node 之间所有通信都必须加密。

包括：

- USB-C 有线同步
- 内部 Wi-Fi
- LoRa
- 蓝牙
- 低速无线

禁止因为有线或内网就裸传。

## D004 - Mac mini 是阶段性 Core

当前以 Mac mini 作为 Core-Proto-01，用于快速开发和验证。它不是最终服务器。

后续会迁移到更低功耗、更坚固、更适合离线部署的设备。

## D005 - 能源优先

所有功能设计都必须考虑功耗与最小可用能力。

## D006 - AI 回答必须符合场景

AI 助手在应急模式下不要给出依赖现代公共服务的建议，例如：

- 联系供水公司
- 联网查询
- 呼叫平台客服
- 依赖云端服务

## D007 - 本地知识库优先

Wiki、应急指南、工具百科、种植百科、地图、手册等内容应尽量本地化。

## D008 - 成员隐私数据必须谨慎

成员信息、医疗、证件、照片等模块必须默认加密，并且开发测试只使用假数据。
