# LanternBox FT-01 v0.2.6a Log Fast UI

基线：v0.2.3 Unified Log/Nav UI。

这版修正 v0.2.3 的问题：

- 不再只是“卡片展示”
- 日志和导航都恢复为“带基本信息的列表”
- 操作接回列表选择和播放 / 打开功能
- 视觉风格继续向路径功能统一

## 日志页

显示最近录音列表，每行包含：

```text
选中标记
序号
文件名
时长
```

按键：

```text
< / >      选择录音
P / Enter  播放选中录音
R          开始录音
S          停止保存
A          刷新列表
U / D      调整回听音量
ESC        返回首页
```

## 导航页

轨迹列表每行包含：

```text
选中标记
序号
session id
路径点数量
最后时间
```

轨迹总览每行包含：

```text
选中标记
点序号
时间
经纬度摘要
```

按键保持：

```text
< / >      选择
Enter      打开轨迹
R          刷新
L          返回轨迹列表
M          地图
N          指南
B          指向基地
ESC        返回首页
```

## 没有修改的稳定底层

```text
录音：Mic Task -> FreeRTOS Queue -> SD Writer Task -> WAV
播放：SD 双缓冲预读 -> Speaker.playRaw -> 播放增益
```

## v0.2.5b 修复

修复插着 LoRa/GNSS Cap 时，进入音频列表 / 回听 / 保存后可能再次出现 SD 读取失败的问题。

原因：

- 主系统 `initSD()` 已经有 LoRa NSS 拉高保护。
- 但 AudioLogger 后续会自己调用 `SD.exists()` / `SD.open()` / `SD.read()` / `SD.write()`。
- 这些运行期 SD 操作之前没有重新确认共享 SPI 总线状态。

本版新增：

```cpp
AudioLogger::prepareSdBus()
```

每次 AudioLogger 访问 SD 前都会重新设置：

```cpp
LORA_NSS_PIN = OUTPUT HIGH
LORA_RST_PIN = OUTPUT HIGH
SD_SPI_CS_PIN = OUTPUT HIGH
```

不修改：

- 录音 RTOS Queue 架构
- 播放双缓冲预读
- 播放增益
- 列表 UI
- GNSS 摘要日志

## v0.2.6a 修复

修复日志页卡顿问题。

原因：

之前每次刷新日志页面都会：

- 读取每一行音频列表
- 打开 SD
- 读取 index.jsonl
- 读取 WAV 文件大小

导致：

- 调整音量需要等待
- 左右选择卡顿
- 页面刷新慢

本版：

- 音频时间 / 日期 / 时长改为缓存。
- 进入日志页只读取一次。
- A 刷新列表时重新建立缓存。
- 普通 UI 操作不再访问 SD。

同时：

- 删除右上角多余 `|` 分隔线。
- 列表显示：
  ```text
  日期  时间  时长
  ```
